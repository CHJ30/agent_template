# 需求规范标准 v1.0

## 完整性检查项

一份合格的需求单必须包含以下字段：

1. **需求单号**：格式 REQ-YYYY-NNN
2. **标题**：简洁描述核心功能（≤20字）
3. **描述**：详细说明功能细节
4. **优先级**：高 / 中 / 低
5. **约束条件**：至少一条，使用"必须 / 至少 / 不得 / 不能"等明确用词
6. **相关实体**：业务名词列表（至少两个）
7. **干系人**：至少列出一个团队或角色
8. **验收标准**：至少一条具体可测试的条件

## 常见缺陷

- 缺少验收标准（最常见）
- 约束条件模糊，避免"尽量"、"可能"、"建议"等词汇
- 未列出相关干系人
- 描述与标题重复，没有提供额外细节

## 完整性评分标准

- 8项全部具备：完整（优秀）
- 6-7项：基本完整（良好）
- 4-5项：不完整（需补充）
- ≤3项：严重缺失（需重写）
